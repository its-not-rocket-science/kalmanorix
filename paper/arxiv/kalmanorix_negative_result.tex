\title{Negative Result Preprint:\\
No Reliable Kalman-over-Mean Gain in Domain-Balanced Retrieval}
\author{Paul Schleifer}
\date{May 2026}

\begin{abstract}
We report a reproducible negative result for mixed-domain retrieval fusion. In the canonical domain-balanced C100 run ($n_{\text{pairs}}=1193$), Kalman-minus-Mean $\Delta$ nDCG@10 is $-9.258801070226193\times10^{-6}$ with Holm-adjusted $p=1.0$; $\Delta$ recall@100 is 0.0. Compute overhead is limited (latency ratio 1.0722551296204925; FLOPs ratio 1.0). The benchmark verdict is \texttt{inconclusive\_sufficiently\_powered}. This preprint emphasizes the empirical outcome and exact reproduction steps.
\end{abstract}

\begin{tcolorbox}[colback=gray!8,colframe=black,title=Scope of conclusion]
Under the tested protocol (domain-balanced mixed-domain benchmark, \texttt{max\_candidates=100}, deterministic embeddings), Kalman-style uncertainty fusion did \emph{not} show a reliable practical improvement over mean fusion.

This does not imply universal inferiority of Kalman methods or universal optimality of mean fusion.
\end{tcolorbox}

\section{Positioning statement}
This arXiv preprint reports one benchmark run and shows how to reproduce it. We focus on what the run produced: metric deltas, significance results, latency, FLOPs, and artefact files. We do not extend the conclusion beyond this configuration. The reporting stance is intentionally aligned with reproducibility checklists for ML experiments \cite{pineau2021improving} and with publication-bias concerns that motivate explicit dissemination of negative findings \cite{fanelli2012negative}.

\section{Setup}
Dataset bundle: \texttt{mixed\_beir\_v1.2.0}. The heterogeneous benchmark framing follows BEIR-style cross-domain retrieval evaluation \cite{thakur2021beir}, with dense sentence embeddings motivated by Sentence-BERT \cite{reimers2019sentencebert}. Split sizes are train=5078, validation=634, and test=1193. Each domain contributes at least 126 test queries. Benchmark configuration uses \texttt{domain-balanced=true} and \texttt{max\_candidates=100}.

Primary endpoint is paired $\Delta$ nDCG@10 (Kalman minus Mean), where nDCG is a ranked-retrieval effectiveness metric \cite{jarvelin2002}. Analysis is paired at the query level and reports Wilcoxon signed-rank testing \cite{wilcoxon1945} with Holm multiple-comparison correction \cite{holm1979}, consistent with IR evaluation guidance on paired significance testing \cite{smucker2007comparison} and broader recommendations for robust statistical comparisons \cite{dror2018hitchhikers,demsar2006statistical}. Predeclared decision thresholds are $|\Delta|\ge 0.02$, Holm-adjusted $p<0.05$, latency ratio $\le 1.5$, and FLOPs ratio $\le 1.1$. Bootstrap uncertainty summaries use the nonparametric bootstrap framework \cite{efron1979bootstrap}.

\section{Main negative result}
The final benchmark artefacts are \texttt{results/evidence\_registry.json} and \texttt{results/claim\_gate.json}. They report:
\begin{itemize}
  \item $\Delta$ nDCG@10 $=-9.258801070226193\times10^{-6}$
  \item Holm-adjusted $p=1.0$
  \item $\Delta$ recall@100 $=0.0$
  \item Latency ratio (Kalman/Mean) $=1.0722551296204925$
  \item FLOPs ratio (Kalman/Mean) $=1.0$
  \item Verdict: \texttt{inconclusive\_sufficiently\_powered}
\end{itemize}

On this test set, Kalman fusion did not beat mean fusion by the predeclared margin.

\section{Interpretation and limits}
\textbf{Supported by this run}
\begin{itemize}
  \item On 1193 paired test queries with \texttt{max\_candidates=100}, Kalman fusion did not meet the effect or significance thresholds.
  \item Mean fusion matched or exceeded Kalman on the tracked outputs: nDCG@10 and recall@100.
  \item Runtime overhead was small in this run (latency ratio 1.0723; FLOPs ratio 1.0).
\end{itemize}

\textbf{Not supported by this run}
\begin{itemize}
  \item This run does not show that Kalman-style fusion is always worse.
  \item This run does not test other embedding models, other domain mixes, larger candidate pools, or alternative uncertainty estimators.
\end{itemize}
